定位和浮动

css 有三种基本的定位机制：普通流，浮动，绝对定位。
默认都存在普通流中，按照元素的规则去排布
一般使用定位指使用position属性, absolute fixed relavive static
static 默认正常
relavive 相对定位，相对现在位置定位
absolute 绝对定位，相对设置定位的父级元素定位，一直寻找到body元素，
使用absolute会使元素脱离文档流
fixed 相对浏览器窗口定位

浮动利用了float属性，达到更好的布局。使用float除了使元素脱离文档流，也产生了一些其他对兄弟元素的影响。

高度塌陷

使用position：absolute或float属性脱离文档流就容易使父元素产生高度塌陷
解释：
在文档流中，父元素若不设置高度，其默认是被子元素撑开的。但是当子元素设置浮动之后，子元素会完全脱离文档流，此时将会导致子元素无法撑起父元素的高度，导致父元素的高度塌陷。

user agent stylesheet

就是浏览器对元素设置的默认样式
如 display: block 是浏览器赋予 div 的，存在于浏览器的 user agent stylesheet ，而不是 div 默认 display 的值就为 block ，在 W3C 中，所有的 HTML 标签 display 的默认值都为 inline 。

可替换元素和非替换元素

可替换元素是浏览器根据标签的元素和属性来判断显示具体内容
典型的:

  1. img 标签 根据标签的src属性值来读取图片信息并显示
  2. input 标签 读取type属性来显示样式

img、input、textarea、select、object 都是替换元素，没有实际的内容(空元素)
其他大多数元素是不可替换元素，直接显示

替换元素，可以设定宽高，如img不设置宽高就默认使用图片的尺寸

块级元素和行内元素

  1. 块级元素 独占一行
  2. 行内元素 和其他行内元素共用一行，几乎所有的可替换元素都是行内元素
