cookie和session

发表于 2017-04-13 | 分类于 js相关

由于http是一种无状态协议，所以需要记录它的连接以便实现一些功能典型的如：
登录，购物车

两种方法的主要的区别是
cookie是一种客户端的方案，session是一种服务器的方案

cookie 是将一些信息保存在了客户端，
限制大小为4k，
一般当浏览器关闭时，当前cookie就会清空
浏览器中可以调试看到cookie，所以不是很安全

session 将信息保存在服务器端
当浏览器第一次发送请求时，服务器自动生成了一个Session和一个Session ID用来唯一标识这个Session，并将其通过响应发送到浏览器。
当浏览器第二次发送请求，会将前一次服务器响应中的Session ID放在请求中一并发送到服务器上，服务器从请求中提取出Session ID，并和保存的所有Session ID进行对比，找到这个用户对应的Session。

一般服务器会在一段时间内保存session ，时间到了，就会销毁
也可使用数据库持久话保存session

一般使用中可以将session ID作为cookie保存在客户端结合使用

MongoDB使用

发表于 2017-04-06 | 分类于 js相关

MongoDB 使用c++语言编写的NoSql(‘不仅仅是sql’)数据库之一，它提出非关系型数据库。最大的特点是高度扩展和灵活。其数据类似于json。

安装

进入官网下载安装包进行安装。
在计算机中一个盘根目录下新建一个如c:data/db文件作为数据库数据文件。
进入目录下使用命令行输入 mongod 显示 ok 说明数据库成功运行。
打开另一个命名行 输入mongo 即可以进入操作行。

基本命令

show dbs 查看所有数据库以及使用量，local数据库是系统本身创建。
use xxx(数据库名) 显示 switched to db xxx 选择使用哪一个数据库
show collections 显示当前数据库中的集合
db.xxx.drop() 删除xxx集合
db.dropDatabase() 删除当前数据库
其他增删改查类似其他数据库

node.js简单配置

发表于 2017-04-02 | 分类于 js相关

Node.js安装

Node.js是使用javaScript语言写的服务端框架。
进入nodejs官网，下载最新的安装程序，按步骤安装到计算机中。
安装完成使用gitbush

可以显示出版本号说明安装成功。
官网经典的服务器创建，新建文件server.js

命令行中运行 node server.js，打开localhost：8080，可以看到返回到的数据。一个简单的node服务器成功运行。

NPM

npm是node包管理工具，随node安装入计算机。
平时开发可以通过npm安装一些依赖，进入npm查看多的包。
使用npm

安装的依赖都将自动放入到项目的node_modules 目录中
项目中生成的package.json 包含了依赖的详细信息。
